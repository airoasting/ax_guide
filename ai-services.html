어떤 AI 서비스를 사용해야 할까요?

첫 분기의 비용·속도·보안을 결정하는 네 가지 질문, 빅3 분담·OpenRouter 활용·자체 LLM 검토·Enterprise 도입 시점을 한 장에 정리합니다.

결론부터 말씀드립니다. 빅3는 셋 다 도입해 사람마다 작업에 맞는 모델을 직접 고르도록 자율을 부여하는 편이 가장 효과적입니다. 분류, 요약, 검색처럼 정형화된 대량 작업은 OpenRouter로 저가 모델에 보내 비용을 낮춥니다. 보안이 중요한 업무에는 빅3의 Enterprise 요금제를 적용하고, 외부 송신 자체가 부담스러운 데이터는 AWS Bedrock, GCP Vertex AI, Azure OpenAI의 VPC 직접 호출로 한 단계 더 격리합니다. 자체 LLM 구축은 데이터의 외부 송신이 원천 금지된 경우, 또는 월 토큰 비용이 자체 서빙의 손익분기점을 넘긴 경우에만 검토합니다. 한 모델로 통일하려는 시도는 대부분 잘못된 답입니다.
빅3 비교 OpenRouter 자체 LLM 보안과 Enterprise 상황별 선택 참고 자료

ChatGPT · Claude · Gemini, 세 모델은 같은 종(種)이 아닙니다

OpenAI · Anthropic · Google · 2026년 1분기 기준

세 모델 모두 범용 대화형 모델이지만, 잘하는 영역과 약한 영역이 분명히 다릅니다. 한 사람에게 한 가지 모델만 제공하면 그 모델의 약점이 곧 사용자 결과물의 한계로 이어집니다. 임원이 던져야 할 질문은 "어느 모델이 가장 똑똑한가"가 아니라 "어느 자리에 어느 모델을 배치할 것인가"입니다.

한국어 표현이 다소 기계적이며 추론의 일관성에 편차가 있습니다. 독립 제품보다 Workspace 내 부가 기능으로 활용되는 경향이 있어 단독 사용 시 경험이 다소 부족합니다.

긴 영상과 PDF 분석, 사내 Workspace 기반 자동화, 시장 조사와 뉴스 큐레이션, 영상·이미지 멀티모달 분석.
"통일"이 아니라 "역할 분담"이 답입니다
한 사람에게 한 가지 모델만 제공하면 그 모델의 약점이 곧 사용자의 한계가 됩니다. 임원, 기획, 법무, 엔지니어가 동일한 모델로 일할 경우, 가장 비싼 자리에서도 가장 저렴한 모델 수준의 정확도밖에 얻지 못합니다. 권장안은 빅3를 모두 도입하고, 작업의 성격에 따라 사용자가 직접 모델을 선택하도록 자율성을 부여하는 것입니다. 1인당 월 60달러 안팎의 추가 비용은 한 번의 의사결정 품질 개선만으로 충분히 회수됩니다. 본격 운영 단계에서는 OpenRouter로 모델 라우팅을 자동화하여, 사용자가 매번 직접 선택하지 않아도 되도록 구성합니다.

OpenRouter · 한 API로 100개 모델을 부르는 모델 게이트웨이

멀티 프로바이더 · 가격 비교 · 자동 폴백

OpenAI, Anthropic, Google은 물론 Mistral·DeepSeek·Qwen·Llama 계열의 오픈웨이트 모델, xAI와 Cohere의 자체 모델까지 단일 엔드포인트로 호출할 수 있는 라우팅 서비스입니다. 운영 관점에서 네 가지 장점이 있습니다.

측면OpenRouter가 푸는 자리현장 적용 예
단일 API 모델사마다 다른 SDK, API 키, 요청 포맷의 차이를 단일 인터페이스로 통합합니다. 모델 교체에 필요한 작업이 코드 한 줄 수준으로 단순화됩니다. 동일한 에이전트 코드로 GPT-5.1, Claude Sonnet 4.6, Gemini 3 Pro를 작업 종류에 따라 교체합니다.
가격 가시화 모델별 입력·출력 토큰 단가가 동일 화면에 표시되어, 모델 선택을 즉석에서 결정할 수 있습니다. 분류와 요약 작업은 Llama 3.3 70B, DeepSeek V3, Qwen 2.5처럼 약 1/10 가격의 모델로 대체합니다.
자동 폴백 1순위 모델에 장애가 발생하면 2·3순위 모델로 자동 재시도합니다. 단일 모델사 장애가 회사 운영 전체로 확산되지 않습니다. Claude API 장애 시 동일 작업을 GPT-5.1로 자동 재시도하도록 폴백 체인을 사전 설정합니다.
저가 모델 활용 오픈웨이트 모델(Llama, Mistral, DeepSeek, Qwen)을 활용해 대량 작업의 단위 비용을 큰 폭으로 절감할 수 있습니다. 하루 1만 건의 분류·태깅·요약 작업은 저가 모델로, 하루 100건의 임원 보고서는 Opus 또는 Gemini 3 Pro로 분리해 처리합니다.
한 가지 유의 사항 · 데이터 흐름
OpenRouter는 게이트웨이 구조이므로, 모든 요청이 OpenRouter 인프라를 한 번 경유합니다. 회사 데이터를 외부로 반출할 수 없는 업무에는 그대로 적용하기 어렵습니다. 이러한 업무는 뒤에서 다룰 Enterprise 요금제 또는 Bedrock, Vertex AI, Azure OpenAI의 직접 호출 방식으로 전환합니다. OpenRouter는 보안 등급이 낮은 일반 업무, 외부 정보 처리, 실험과 시제품 단계에 적합합니다.

자체 LLM 구축은 언제 진지하게 검토하는가

Llama · Qwen · Mistral · vLLM · NVIDIA NIM · H100

"우리 회사만의 AI를 만들고 싶다"는 요구사항은 대부분 외부 모델로 해결할 수 있습니다. 다만 아래 조건 중 두 가지 이상이 동시에 해당하는 경우는 예외입니다. 이 시점부터는 외부 모델로 해결되지 않으므로 자체 서빙을 진지하게 검토할 단계입니다.

외부 API로 충분한 경우

대부분의 일반 업무가 이 범주에 해당합니다
  • 월 토큰 사용량이 수억 원 이하인 회사. 자체 서빙의 고정비(GPU, 인력, 인프라)가 외부 API의 변동비를 회수하기 어렵습니다.
  • Enterprise 요금제와 VPC 호출로 보안 요건이 해결되는 회사. 학습 미사용, SSO, VPC만 확보해도 대부분의 요건이 충족됩니다.
  • 최신 모델의 추론 성능을 분기마다 반영해야 하는 회사. 자체 모델로는 빅3의 분기 단위 업그레이드 속도를 따라가기 어렵습니다.
  • 고품질 도메인 데이터가 충분하지 않은 회사. 데이터가 부족한 상태에서의 fine-tuning은 오히려 성능 저하를 초래합니다.
  • GPU 운영과 MLOps 인력이 부족한 회사. 모델 배포는 시작 단계일 뿐이며, 패치·평가·롤백을 주 단위로 수행해야 합니다.

자체 LLM을 검토해야 하는 경우

아래 조건 중 두 가지 이상이 동시에 해당해야 검토 대상이 됩니다
  • 데이터의 외부 송신이 원천 금지된 경우. 규제, 계약, 고객 합의로 외부 호출 자체가 불가능한 환경(국방, 일부 금융, 일부 의료).
  • 월 토큰 비용이 자체 서빙의 손익분기점을 초과한 경우. 월 청구액이 H100 8대 운영비를 초과하는 수준이라면 ROI가 자체 서빙 쪽으로 기울기 시작합니다.
  • 도메인 특화 fine-tuning의 가치가 명확한 경우. 고품질 자사 데이터로 학습할 경우 외부 모델 대비 정확도가 유의미하게 향상됩니다.
  • 저지연 응답이 결정적인 경우. 외부 API의 네트워크 왕복 지연으로는 요구 수준을 충족할 수 없는 실시간 추론(자율주행, 실시간 통제).
  • 장기적 모델 자주권이 전략 자산인 경우. 외부 모델사에 대한 의존도 자체가 산업 내 협상력으로 환산되는 분야.

자체 서빙으로 방향을 잡았다면, 표준 스택은 다음과 같습니다. 모든 옵션을 동시에 검토하기보다는 한 가지를 깊이 있게 진행하는 편이 효율적입니다.

자체 LLM 운영 스택의 표준 조합
모델
오픈웨이트 베이스
Llama 3·4, Qwen 2.5·3, Mistral, DeepSeek V3가 1차 후보군입니다. 한국어 처리 성능은 Qwen과 DeepSeek 계열이 우수하며, 라이선스 안정성은 Llama가 가장 검증되어 있습니다. 베이스 모델 선정 후 자사 데이터를 활용해 LoRA 또는 QLoRA 방식으로 fine-tuning을 수행합니다.
서빙
vLLM · TGI · NVIDIA NIM
vLLM이 오픈소스 표준으로 가장 널리 채택되어 있습니다. 운영 안정성과 엔터프라이즈 지원을 동시에 확보하려면 NVIDIA NIM 마이크로서비스 또는 Hugging Face TGI를 검토합니다.
인프라
H100·H200 GPU 클러스터
70B 클래스 모델 1대 추론에는 H100 4~8장이 필요합니다. 자체 데이터센터 구축은 12~18개월의 리드 타임이 소요되므로, 통상 AWS·GCP·Azure의 GPU 임대 또는 CoreWeave·Lambda 등 GPU 전문 클라우드로 시작합니다.
운영
평가·관측·재학습 루프
자체 평가 데이터셋(eval set) 보유가 모델 학습 자체보다 더 중요합니다. 회사 도메인의 정답 기준이 없으면 모델 성능의 개선 여부를 판단할 수 없습니다. 관측 도구로는 LangSmith, Langfuse, Arize, WhyLabs가 표준 후보군입니다.
자체 LLM에 대한 가장 흔한 오해
"보안 때문에 자체 모델을 구축한다"는 결론은 대부분 한 단계 성급한 판단입니다. 보안 요건의 90퍼센트는 Enterprise 요금제와 Bedrock, Vertex AI, Azure OpenAI의 VPC 호출로 충분히 해결됩니다. 자체 LLM이 정답이 되는 경우는 세 가지로 정리됩니다. 데이터의 외부 송신이 원천 금지된 경우, 토큰 비용이 자체 서빙의 손익분기점을 초과한 경우, 도메인 특화 fine-tuning의 가치가 검증된 경우입니다. 이 세 가지가 함께 충족되지 않으면, 자체 모델은 빅3 대비 비용과 속도 모두에서 불리합니다.

보안 요건이 높을수록 Enterprise 요금제, 그 다음은 클라우드 직접 호출

학습 미사용 · SSO · DLP · VPC 호출

법무, 재무, HR, 고객 데이터처럼 외부 학습에 활용되어서는 안 되는 데이터는 무료나 일반 요금제로 처리하지 않습니다. 두 단계의 선택지가 있습니다. 1단계는 빅3의 Enterprise 요금제이고, 2단계는 AWS Bedrock, GCP Vertex AI, Azure OpenAI를 통한 직접 호출입니다.

플랫폼학습 미사용인증·통제특징
ChatGPT Enterprise · Team 기본 미사용 SSO(SAML), SCIM, DLP, 감사 로그 SOC 2 Type II, GDPR 인증을 보유합니다. 가장 광범위한 사용 기반과 가장 완성도 높은 관리 콘솔을 제공합니다.
Claude Enterprise · Team 기본 미사용 SSO, SCIM, 감사 로그, 도메인 캡처 SOC 2 Type II와 HIPAA(BAA)를 지원합니다. 긴 컨텍스트, 코딩, 문서 작업이 핵심인 업무에 적합합니다.
Gemini for Workspace · AI Ultra 기본 미사용 Google Workspace 관리 콘솔, DLP, Vault Workspace를 이미 도입한 회사라면 추가 도입 비용이 거의 발생하지 않습니다.
AWS Bedrock 모델 사용 데이터 학습 없음 VPC 내 호출, IAM, KMS 키 관리, PrivateLink Claude, Llama, Mistral, Titan 등 다수 모델을 동일한 거버넌스 체계에서 운영할 수 있습니다.
GCP Vertex AI 모델 사용 데이터 학습 없음 VPC Service Controls, CMEK, IAM Gemini, Claude, Llama를 모두 지원하며, BigQuery 및 Workspace 데이터와의 연동성이 우수합니다.
Azure OpenAI 입출력 데이터 학습 미사용 Private Endpoint, Entra ID, 콘텐츠 필터 Microsoft 365와 Azure를 이미 활용 중인 회사의 1순위 옵션이며, 한국 리전(서울)도 제공됩니다.
의사결정 순서
첫 단계로 실험과 시제품은 일반 요금제를 사용해 빠르게 운영합니다. 다음 단계로 팀 단위 운영이 시작되는 시점에 Team 또는 Enterprise 요금제로 전환해 학습 미사용, SSO, 감사 로그를 확보합니다. 마지막 단계로 고객, 재무, 계약처럼 외부 반출이 허용되지 않는 데이터는 Bedrock, Vertex AI, Azure OpenAI의 VPC 호출로 한 단계 더 격리합니다. 이 단계를 건너뛸 경우, 보안 사고가 발생하기 전까지는 비용만 증가하고, 사고가 발생한 뒤에야 비로소 그 비용이 정당화됩니다.

상황별 권장 조합

개인 · 팀 · 대기업 · 보안 민감 업무 · 데이터 격리

회사 규모, 보안 요건, 워크로드 성격에 따라 권장 조합이 달라집니다. "모델 하나만 선택해 달라"는 질문에는 답하기 어렵지만, "현재 어느 위치에 있는가"라는 질문에는 명확히 답할 수 있습니다.

스택
이유
개인 · 1인 실험가
ChatGPT Plus + Claude Pro를 병행하고, 필요한 경우 Gemini Free를 보조로 사용합니다.
월 40달러로 빅3의 강점을 모두 확보할 수 있습니다. 특정 모델에 종속되지 않는 감각을 가장 저렴하게 습득하는 단계입니다.
스타트업 · 10~50인 팀
사내 사용에는 ChatGPT Team + Claude Team을, 백오피스 자동화에는 OpenRouter를 적용합니다.
두 빅3로 사내 사용을 분담하고, 대량 처리는 OpenRouter로 저가 모델까지 활용합니다. 토큰 비용이 인건비 대비 결정적이지 않은 구간입니다.
중견·대기업 · 일반 업무
두 가지 Enterprise 요금제(ChatGPT Enterprise + Claude Enterprise)에 Gemini for Workspace를 결합합니다.
SSO, 감사 로그, 학습 미사용이 기본 요건입니다. Workspace를 이미 도입한 회사라면 Gemini는 추가 비용 없이 사용할 수 있습니다.
대기업 · 보안 민감 업무
AWS Bedrock(Claude, Llama, Mistral), Vertex AI(Gemini, Claude), Azure OpenAI(GPT) 중 한 가지를 선택합니다.
VPC 호출과 KMS 키 관리로 데이터가 회사 클라우드를 벗어나지 않습니다. 현재 사용 중인 퍼블릭 클라우드 사업자를 1순위 후보로 검토합니다.
데이터 외부 송신 금지
자체 서빙(Llama, Qwen, Mistral) 기반에 vLLM 또는 NIM, 그리고 자체 평가셋을 구성합니다.
규제나 계약상의 이유로 외부 호출 자체가 불가능한 경우입니다. 모델 자주권이 운영의 전제 조건이 되는 환경이며, 빅3는 비공식 정보 조회 용도로만 활용합니다.
조합은 분기마다 재점검합니다
빅3는 분기 단위로 모델을 교체하고 가격을 절반 수준으로 인하합니다. 6개월 전의 정답이 현재의 정답이 아닐 수 있다는 의미입니다. 자산 축적 4단계 순환의 검증 단계에서 사용량, 만족도, 비용을 분기마다 재집계하고, 그 데이터를 바탕으로 다음 분기의 모델 조합을 다시 설계합니다. 한 번 결정한 스택을 1년 동안 그대로 유지하는 것이 가장 비싼 의사결정입니다.
